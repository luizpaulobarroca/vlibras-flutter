**Step 2 — lib/src/vlibras_js.dart:** Migrate JS interop bindings from spike. Exact content:

```dart
import 'dart:js_interop';
import 'dart:js_interop_unsafe';

@JS('VLibras')
external VLibrasNamespace get vLibras;

extension type VLibrasNamespace._(JSObject _) implements JSObject {
  external JSFunction get Player;
}

extension type VLibrasPlayerOptions._(JSObject _) implements JSObject {
  external factory VLibrasPlayerOptions({
    String translator,
    String targetPath,
  });
}

extension type VLibrasPlayerInstance._(JSObject _) implements JSObject {
  external void load(JSObject element);
  external void translate(String text);
  external void pause();
  external void stop();
  @JS('continue') external void resume();
  external void repeat();
  external void setSpeed(double speed);
  external void on(String event, JSFunction callback);
  external void off(String event, JSFunction callback);
}

VLibrasPlayerInstance createVLibrasPlayer() {
  final options = VLibrasPlayerOptions(
    translator: 'https://vlibras.gov.br/api',
    targetPath: '/vlibras/target',
  );
  return vLibras.Player.callAsConstructor(options) as VLibrasPlayerInstance;
}
```

Key details: `@JS('continue')` on `resume()` (continue is a Dart keyword). Both `dart:js_interop` AND `dart:js_interop_unsafe` required — `callAsConstructor` is in `dart:js_interop_unsafe`. `targetPath` fixed at `/vlibras/target`, `translator` at `https://vlibras.gov.br/api` — locked in CONTEXT.md.

**Step 3 — lib/src/vlibras_web_platform.dart:** Implement VLibrasWebPlatform. Key design:
- Constructor: `VLibrasWebPlatform({required void Function(VLibrasStatus) onStatus, Duration timeout = const Duration(seconds: 30)})`
- Fields: `final void Function(VLibrasStatus) onStatus`, `final Duration timeout`, `VLibrasPlayerInstance? _player`, `Completer<void>? _translateCompleter`, `Timer? _timeoutTimer`, `Completer<void>? _initCompleter`
- `attachToElement(web.HTMLDivElement div)`: called by VLibrasView. Creates player via `createVLibrasPlayer()`, calls `_registerEvents(player)`, then `player.load(div as JSObject)`, stores in `_player`.
- `_registerEvents(player)`: registers `player.on('load', _onLoad.toJS)`, `player.on('animation:play', _onPlay.toJS)`, `player.on('animation:end', _onEnd.toJS)`. ALL callbacks MUST use `.toJS` — omitting causes runtime TypeError.
- `_onLoad()`: completes `_initCompleter` if not null and not completed.
- `_onPlay()`: calls `onStatus(VLibrasStatus.playing)`.
- `_onEnd()`: calls `onStatus(VLibrasStatus.ready)`, then calls `_completeTranslate(null)`.
- `_completeTranslate(Object? error)`: cancels `_timeoutTimer`, sets it null, pops `_translateCompleter` to local var and nulls the field, completes it (with error if non-null, otherwise successfully). Guards with `!c.isCompleted`.
- `initialize()`: creates `_initCompleter = Completer<void>()`, returns its future. `_player` is not available yet (View hasn't called `attachToElement`). If `_initCompleter` already set, return existing future (idempotent).
- `translate(String text)`: cancel-and-restart — call `_completeTranslate(Exception('cancelled'))` first. Create new `_translateCompleter`, set `_timeoutTimer = Timer(timeout, ...)`. Timer callback: `_completeTranslate(TimeoutException('animation:end not received within $timeout', timeout))`. Then `_player!.translate(text)`, return `_translateCompleter!.future`.
- `pause()`, `stop()`, `resume()`, `repeat()`, `setSpeed(double)`: delegate directly to `_player` (simple one-liners).
- `dispose()`: call `_completeTranslate(Exception('disposed'))`, `_initCompleter?.complete()` safely, set `_player = null`.

Imports needed: `dart:async`, `dart:js_interop`, `package:web/web.dart as web`, `vlibras_js.dart`, `../vlibras_platform.dart`, `../vlibras_value.dart`.

Note: `VLibrasWebPlatform` does NOT implement `initialize()` by wiring to VLibrasView. The flow is: controller calls `initialize()` which sets up the init completer; separately VLibrasView calls `attachToElement()` on the platform (via `controller.attachElement()`), which fires `player.load()`, which eventually triggers `_onLoad()`, which completes the init completer.
