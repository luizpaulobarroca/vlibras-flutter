<objective>
Fix the pubspec.yaml regression introduced by plan 04-05 that broke all 40 VM tests.

Purpose: Plan 04-05 added `flutter: plugin: platforms: web: {}` to satisfy pana platform scoring. This form is invalid for flutter_tools: WebPlugin.fromYaml requires either a `pluginClass` or `dartPluginClass` field when a `platforms:` block is declared. Without it, flutter_tools throws "The plugin vlibras_flutter is missing the required field pluginClass" and all 40 VM tests fail to load.

The fix is one line: change `web: {}` to `web: { dartPluginClass: null }`. This signals that the package intentionally has no Dart plugin class (it uses conditional imports, not a plugin registrar). This satisfies both pana platform scoring (block is present) and flutter_tools (field is present, value null is the conventional sentinel for absence).

Output: pubspec.yaml with `web: { dartPluginClass: null }`, all 40 VM tests passing, dry-run still clean.
</objective>
